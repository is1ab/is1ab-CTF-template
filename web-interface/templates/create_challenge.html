創建挑戰

與命令列 scripts/create-challenge.py 二選一，皆需出題人、驗題人、類型與難度等資訊

建議流程

  1. 先設定團隊：在專案根目錄 config.yml 填寫 team.default_author（預設出題人），並依競賽計畫調整 challenge_quota（各類型／難度數量）。
  2. 本頁建題：依序填寫題名、說明、類別、難度、分數、出題人、驗題人、題目類型；分數可隨難度自動帶入。
  3. 驗題：到「驗題」分頁對題目執行通過／退回；通過前請在本地實作並跑 make validate 等檢查。

完整雙管道說明見專案 docs/authoring-challenges.md。

📁 目錄結構

創建成功後，請將題目相關檔案放置到以下目錄：

  • src/ - 題目原始碼（完整的題目實現）
  • files/ - 提供給參賽者下載的檔案
  • writeup/ - 解題說明文檔
  • docker/ - Docker 部署相關檔案
  • bin/ - 編譯後的可執行檔案
💡 重要提醒
  • 🔒 src/ 目錄中的原始碼不會提供給參賽者
  • 📦 只有 files/ 目錄中的檔案會提供下載
  • 🐳 Web/PWN 類題目建議使用 Docker 部署
  • 📝 請務必在 writeup/ 中撰寫詳細解題步驟
  • 🔍 可以在題目詳情頁面查看完整的檔案結構
